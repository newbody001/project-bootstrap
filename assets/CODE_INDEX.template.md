## 架构依赖关系

> 当需要**换框架、换交互模式、换数据库、重写子系统**时，
> 先看这张表，了解哪些文件会受影响、哪些完全安全。

### 核心依赖链

```
入口文件 (app.py / main.py / index.js)
    ├── 路由层 → routes/、controllers/
    ├── 业务逻辑层 → services/、game_engine.py
    │       ├── 数据层 → models/、db.py
    │       └── 配置层 → config.py、constants.py
    └── 视图层 → templates/、static/
```

### 影响范围速查

| 如果要换... | 必须改的文件 | 可能受影响 | 不受影响 |
|-----------|------------|-----------|---------|
| 后端框架 (Flask→FastAPI) | `app.py`、所有 `routes/` | `session_manager.py`、中间件 | `models/`、`templates/`、`static/` |
| 前端渲染 (Jinja2→SPA) | `templates/*`、`app.py` 的页面路由 | API 路由（保持 JSON 则安全） | `models/`、业务逻辑层 |
| 数据库 (SQLite→PostgreSQL) | `config.py` 的连接串、`models/` 的方言差异 | 无 | `routes/`、业务逻辑层 |
| 会话管理 (localStorage→JWT) | `session_manager.py`、`app.py` 的鉴权中间件 | 所有路由的 session 读取点 | 纯业务逻辑、`constants.py` |

## 关键注意事项

> 容易踩坑的点，Agent 开始改代码前必须知道。

1. **配置文件**：数据库连接串在 `config.py` 的 `DATABASE_URL` 变量，不要在其他文件硬编码
2. **编码约定**：Python 文件统一 `# -*- coding: utf-8 -*-`，缩进 4 空格
3. **模板路径**：Jinja2 模板在 `templates/` 目录，引用时不加前缀
4. **静态资源**：CSS/JS/图片放在 `static/` 目录，模板中用 `{{ url_for('static', filename='...') }}`
